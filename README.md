# Демо сканера маркировки

Мобильная веб-демка для проверки Data Matrix кода маркировки. Первый MVP работает без сборки: HTML, CSS и JavaScript.

## Запуск локально

```bash
python3 -m http.server 5173
```

Открыть на компьютере:

```text
http://localhost:5173
```

Локально можно проверить интерфейс, ручной ввод, demo-коды и парсер.

## Тест с телефона

Камера в мобильном браузере требует secure context. Для телефона лучше открыть демку по HTTPS:

- GitHub Pages;
- Netlify;
- Vercel;
- Cloudflare Pages;
- HTTPS-туннель вроде Cloudflare Tunnel или ngrok.

`http://192.168.x.x:5173` с телефона обычно не даст доступ к камере.

## Быстрая HTTPS-ссылка через Cloudflare Tunnel

Если `cloudflared` уже скачан в `.tools`, запустить локальный сервер:

```bash
python3 -m http.server 5173
```

В другом терминале поднять временную HTTPS-ссылку:

```bash
.tools/cloudflared tunnel --url http://localhost:5173
```

В выводе появится адрес вида:

```text
https://example-example-example.trycloudflare.com
```

Эту ссылку можно открыть на телефоне. Она временная и работает, пока запущены локальный сервер и tunnel-процесс.

## Что уже есть

- мобильный интерфейс;
- ручной ввод кода;
- 4 demo-сценария;
- парсер базовых GS1 Application Identifiers;
- mock-реестр товаров и покупок;
- камера через `getUserMedia`;
- сканирование через `BarcodeDetector`, если браузер поддерживает `data_matrix`;
- загрузка фото как fallback для браузеров с `BarcodeDetector`;
- PWA manifest и service worker.

## Демо-коды

В интерфейсе есть 4 кнопки:

- покупка найдена;
- товар найден, покупки нет;
- код структурно корректный, но не найден;
- некорректный код.

Разделитель GS можно вводить как `<GS>`.

## Ограничения

- `BarcodeDetector` и формат `data_matrix` поддерживаются не во всех мобильных браузерах.
- Дата покупки не берется из самого Data Matrix, а подставляется из mock-реестра.
- Реальная проверка статуса и чековых данных должна идти через backend.
- Секреты, сертификаты и ключи внешних интеграций нельзя хранить во frontend-коде.
